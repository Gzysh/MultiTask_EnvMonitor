# 多任务智能环境监测站 — 框架计划

## 一、项目概述

**项目名称**：多任务智能环境监测站 (Multi-Task Smart Environment Monitor)

**一句话描述**：基于 STM32F103 + FreeRTOS 的多传感器数据采集、显示、存储与报警系统。

**硬件平台**：DShanMCU-F103（STM32F103VET6）
**软件平台**：FreeRTOS V10.0.1 + STM32CubeMX HAL
**开发工具**：Keil MDK-ARM

## 三、任务详细设计

### 3.1 任务总览

| 任务名 | 函数名 | 优先级 | 周期 | 栈大小(words) | 主要功能 |
|--------|--------|--------|------|---------------|---------|
| sensor_task | SensorTask | 2 | 500ms | 256 | 采集所有传感器数据 |
| display_task | DisplayTask | 1 | 消息触发 | 256 | OLED 多页面显示 |
| key_task | KeyTask | 2 | 循环读取 | 128 | 红外遥控输入 |
| alarm_task | AlarmTask | 3 | 事件触发 | 128 | 声光报警 + RGB LED 控制 |
| storage_task | StorageTask | 1 | 5s周期 | 256 | 数据写入 SPI Flash |
| comms_task | CommsTask | 1 | 500ms触发 | 128 | USART1 通信输出 (传感器数据 + 健康报告) |

### 3.2 优先级设计原则

```
优先级高                    alarm_task  (3)
                          sensor_task  (2)  key_task  (2)
优先级低                    display_task(1)  storage_task(1)  comms_task(1)
     ▲
     │
     └── 解释：报警最紧急；传感器和输入其次；
         显示和存储可慢一点，不能阻塞紧急任务
```

### 3.3 sensor_task — 传感器采集

**行为**：
1. 每 500ms 采集一轮所有传感器
2. 将数据打包为 `sensor_data_t`，通过 `xSensorQueue` 发送
3. 同时将数据通过 `xStorageQueue` 发给存储任务，通过 `xUartTxQueue` 发给通信任务
4. 检查阈值，越界则设置 `xAlarmEventGroup` 对应位
5. 无报警时调用 `ColorLED_SetFast(COLOR_GREEN)` 指示系统正常

**伪代码**：
```
SensorTask:
    DHT11_Init()
    LightSensor_Init()
    SR04_Init()

    while (1):
        // 采集数据
        DHT11_Read(&hum, &temp)
        LightSensor_Read(&light)
        SR04_Read(&dist)

        // 打包
        data = {temp, hum, light, dist}

        // 发送给显示任务
        xQueueSend(xSensorQueue, &data)

        // 发送给存储任务(非阻塞)
        xQueueSend(xStorageQueue, &data)

        // 发送给通信任务(非阻塞)
        xQueueSend(xUartTxQueue, &data)

        // 检查阈值
        if temp > TEMP_HIGH_THRESHOLD:
            xEventGroupSetBits(xAlarmGroup, ALARM_TEMP_HIGH)
        if dist < DIST_MIN_THRESHOLD:
            xEventGroupSetBits(xAlarmGroup, ALARM_DIST_NEAR)

        vTaskDelay(pdMS_TO_TICKS(500))
```

### 3.5 key_task — 红外遥控

**行为**：
```
KeyTask:
    IRReceiver_Init()

    while (1):
        if IRReceiver_Read(&dev, &data) == 0:
            // 映射到统一按键码
            cmd = map_ir_code(data)
            // 发给显示任务
            xQueueSend(xKeyQueue, &cmd)
        vTaskDelay(pdMS_TO_TICKS(50))  // 20ms 轮询间隔
```

**红外键码映射**：
| 遥控器按键 | data值 | key_cmd_t |
|-----------|--------|-----------|
| CH- | 0xa0 | KEY_BACK |
| CH | 0xa2 | (保留) |
| CH+ | 0xa4 | KEY_CONFIRM |
| PREV | 0xa8 | KEY_UP |
| NEXT | 0x90 | KEY_DOWN |
| PLAY/PAUSE | 0xae | KEY_CONFIRM |
| VOL- | 0xa6 | KEY_LEFT |
| VOL+ | 0x92 | KEY_RIGHT |
| EQ | 0x9a | KEY_MENU |
| 100+ | 0x9c | (保留) |

### 3.6 alarm_task — 声光报警

**行为**：
```
AlarmTask:
    ActiveBuzzer_Init()
    ColorLED_Init()
    ColorLED_Start()
    上电自检: 蜂鸣器 200ms + RGB 白灯 200ms

    while (1):
        bits = xEventGroupWaitBits(
            xAlarmGroup,        // 事件组
            ALL_ALARM_BITS,     // 等待所有报警位
            pdTRUE,             // 收到后清除
            pdFALSE,            // 任意一个触发即可
            portMAX_DELAY       // 一直等
        )

        if bits & ALARM_TEMP_HIGH:
            蜂鸣器模式播放, RGB 红灯亮
        if bits & ALARM_DIST_NEAR:
            蜂鸣器急促响, RGB 红灯闪烁

        条件清除后: RGB 黄灯 3s 停止阶段
```

### 3.7 storage_task — SPI Flash 存储

**设计决策**：

W25Q64 特性：
- 容量：8MB（64Mbit）
- 页大小：256 字节
- 扇区大小：4096 字节（4KB）
- 擦除寿命：约 10 万次

**存储格式**（循环覆盖，FIFO）：

```
W25Q64 内存布局：
┌──────────────────────────────────────────────────────┐
│  扇区0: 系统信息 (magic + 写指针)    (4KB)           │
├──────────────────────────────────────────────────────┤
│  扇区1~N: 传感器数据记录                              │
│  ┌────────┬────────┬────────┬────────┬────────┐     │
│  │记录1   │记录2   │记录3   │...     │记录N   │     │
│  │(16B)   │(16B)   │(16B)   │        │(16B)   │     │
│  └────────┴────────┴────────┴────────┴────────┘     │
│  每个扇区可存 4096/16 = 256 条记录                   │
│  从扇区1开始写，写满后回到扇区1循环覆盖               │
└──────────────────────────────────────────────────────┘
```

**每条记录 16 字节**：
```c
typedef struct {
    uint32_t timestamp;     // 时间戳(系统 tick)
    int16_t temperature;    // 温度(℃ × 10)
    uint16_t humidity;      // 湿度(% × 10)
    uint16_t light;         // 光照(ADC 原始值)
    uint16_t distance;      // 距离(cm)
    uint8_t reserved[2];    // reserved[0]=record_type, reserved[1]=CRC-8
} __attribute__((packed)) record_t;
```

**CRC-8 校验**：
- 算法：CRC-8-ATM (poly 0x07)，bit-by-bit 计算
- 校验范围：除 reserved[1] 外的全部 15 字节
- 写入：`storage_write_record()` 设置 record_type 后自动计算并写入 reserved[1]
- 读取：`storage_read_newest()` 验证 CRC，旧记录 (reserved[1]==0) 跳过校验以兼容升级

**行为**：
```
StorageTask:
    W25Q64_Init()
    从扇区0读取写指针
    如果 Flash 为空 → 初始化

    while (1):
        result = xQueueReceive(xStorageQueue, &data, pdMS_TO_TICKS(5000))
        if result == pdTRUE:
            // 收到数据，写入缓冲区
            将 data 打包为 record_t 存入缓冲区
        else:
            // 超时(5s 无新数据或周期写入)
            if 缓冲区不为空:
                擦除当前扇区
                W25Q64_Write(当前地址, 缓冲区数据)
                更新写指针到 Flash 扇区0
```

### 3.8 comms_task — 通信输出

**优先级 1**, 阻塞在 `xQueueReceive(xUartTxQueue)` 上。

**行为**：
```
CommsTask:
    while (1):
        xQueueReceive(xUartTxQueue, &data, portMAX_DELAY)

        // 发送传感器文本行 (串口助手直接可读)
        send_text_line(&data)
        → [SENSOR] T:28 H:65 L:1234 D:50\r\n

        // 每 60 次 (~30s) 插入一行健康报告
        计数器++
        if 计数器 >= 60:
            计数器 = 0
            send_health_line()
            → [HEALTH] SensorTask=128 ... HeapFree=4520\r\n
            (使用 uxTaskGetStackHighWaterMark + xPortGetFreeHeapSize)
```

**UART 输出格式**：

| 类型 | 格式 | 频率 |
|------|------|------|
| 传感器 | `[SENSOR] T:<int> H:<int> L:<int> D:<int>\r\n` | 每 500ms |
| 健康 | `[HEALTH] <Task>=<HWM> ... HeapFree=<bytes>\r\n` | 每 ~30s |

## 四、通信接口定义

### 4.1 传感器数据结构体

```c
typedef struct {
    int temperature;        // DHT11 温度 (℃)
    int humidity;           // DHT11 湿度 (%)
    uint32_t light;         // 光敏 ADC 值 (0~4095)
    uint32_t distance;      // 超声波距离 (cm)
    uint8_t  record_type;   // 记录类型: 0=auto,1=manual,2=warning
} sensor_data_t;
```

### 4.2 队列定义

| 队列句柄 | 类型 | 长度 | 发送者 | 接收者 |
|---------|------|------|--------|--------|
| xSensorQueue | sensor_data_t | 3 | sensor_task | display_task |
| xStorageQueue | sensor_data_t | 5 | sensor_task | storage_task |
| xKeyQueue | key_cmd_t | 5 | key_task | display_task |
| xUartTxQueue | sensor_data_t | 3 | sensor_task | comms_task |

### 4.3 事件组定义

```c
#define ALARM_TEMP_HIGH     (1 << 0)   // 温度过高
#define ALARM_TEMP_LOW      (1 << 1)   // 温度过低
#define ALARM_DIST_NEAR     (1 << 2)   // 距离过近
#define ALARM_HUM_HIGH      (1 << 3)   // 湿度过高
#define ALARM_HUM_LOW       (1 << 4)   // 湿度过低
#define ALARM_LIGHT_LOW     (1 << 5)   // 光照过低
```

## 七、Python 上位机工具

### 7.1 概述

上位机位于 `host_tool/` 目录，使用 `pyserial + tkinter + matplotlib`，通过 USART1 接收 MCU 发送的文本帧，实现实时数据可视化。

### 7.2 数据流

```
MCU (USART1 115200)
  │ [SENSOR] T:28 H:65 L:1234 D:50\r\n  (每 500ms)
  │ [HEALTH] SensorTask=128 ... HeapFree=4520\r\n  (每 ~30s)
  ▼
serial_handler (后台线程, readline + 正则解析)
  │ queue.Queue
  ▼
gui_app (主线程, after 100ms 轮询)
  ├── 数值面板: 4 个 LabelFrame 实时显示最新值
  ├── 健康面板: 6 任务栈进度条 + 堆余量
  ├── 曲线面板: matplotlib 4 子图 (温度/湿度/光照/距离), 60s 滚动窗口, 固定 Y 轴量程
  └── CSV 记录: 时间戳文件名, auto-flush 每行
```

### 7.3 使用方式

```bash
pip install pyserial matplotlib
cd host_tool
python main.py                    # 自动扫描 COM 口, 手动连接
python main.py --port COM3        # 指定端口自动连接
```

## 九、面试话术准备（提前规划）

这个项目在面试中可能会被问到的问题：

| 面试问题 | 回答要点 |
|---------|---------|
| "为什么拆成 6 个任务？" | 按并发活动拆分：采集(周期性) / 显示(数据驱动) / 输入(事件驱动) / 报警(紧急) / 存储(后台) / 通信(数据输出) |
| "优先级怎么定的？" | 报警最高(安全)；传感器和输入中等(不能丢数据)；显示和存储最低(可慢) |
| "为什么不用全局变量而用队列？" | 队列保证 ISR 安全、任务间数据拷贝、阻塞等待不浪费 CPU |
| "EventGroup 和信号量的区别？" | EventGroup 可同时等多个标志位，适合多条件报警；信号量适合"一次通知" |
| "SPI Flash 磨损怎么办？" | 循环写入 + 扇区均衡，延长寿命；预留系统信息扇区做日志头 |
| "OLED 访问冲突怎么解决？" | Mutex 保护或单任务独占(本项目把显示集中在 display_task，避免冲突) |
| "UART 协议如何设计的？" | 文本行协议 `[SENSOR] T:xx H:xx ...`，串口助手直接可读；预留二进制帧协议 |
| "上位机如何与 MCU 通信？" | Python 后台线程读串口 + 正则解析，主线程 tkinter 更新；queue.Queue 线程间传递 |
| "如何监控任务栈使用？" | FreeRTOS `uxTaskGetStackHighWaterMark()` API，每 30s 上报 PC 可视化 |
